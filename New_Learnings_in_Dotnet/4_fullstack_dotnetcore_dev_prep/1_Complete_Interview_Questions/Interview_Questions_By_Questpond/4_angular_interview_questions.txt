
Q1. What is Angular and its uses and features ?

Answer:

• Angular is a popular front-end web application framework developed by Google. It’s used to build dynamic, single-page applications (SPAs). 

• Angular is a javascript binding framework (view model) which binds the html (view) ui and javascript (model). This helps you to reduce effort on writting those lenthy lines of code for binding.

Here are its main features explained clearly
--------------------------------------------

1. Component-Based Architecture
Angular applications are built using components.
Each component controls a part of the UI (view + logic), making code modular and reusable.

2. Two-Way Data Binding
Angular automatically keeps data in sync between the model (logic) and the view (UI).
If data changes in one place, it updates everywhere.

3. Directives
Directives extend HTML with custom behavior.
Examples: *ngIf → conditionally display elements
          *ngFor → loop through data

4. Dependency Injection (DI)
Angular has a built-in system to manage services and dependencies.
This makes code easier to test, reuse, and maintain.

5. Modules
Angular organizes code into modules (NgModules).
Each module groups related components, services, and features.

6. Routing
Angular supports client-side routing, allowing navigation between pages without reloading the browser.

7. HTTP Client
Angular provides tools to communicate with backend APIs using HTTP (GET, POST, etc.).

8. TypeScript Support
Angular is built using TypeScript, which adds:
Strong typing
Better tooling (autocomplete, error checking)

9. Forms Handling
Supports both: Template-driven forms (simple forms)
               Reactive forms (complex, scalable forms)

10. Observables & RxJS
Angular uses RxJS (Reactive Extensions) for handling asynchronous data like API calls, events, etc.

11. Built-in Testing Support
Angular supports unit testing and end-to-end testing out of the box.

12. CLI (Command Line Interface)
Angular CLI helps developers: Create projects quickly
                              Generate components/services
                              Build and deploy apps easily

13. Ahead-of-Time (AOT) Compilation
Angular compiles code during build time, which: Improves performance
                                                Reduces runtime errors

14. Cross-Platform Support
Angular can be used to build: Web apps
                              Mobile apps (with Ionic)
                              Desktop apps (with Electron)

Q4. What are type of directives ?

Answer:

There are 3 types of directives in angular.

1). Strutural directives 
These change the structure of the DOM by adding or removing elements.
They usually start with *.
eg :- *ngIf → conditionally shows/hides elements
      *ngFor → loops through a list
      *ngSwitch → switches between views
code eg :- <div *ngIf="isLoggedIn">Welcome!</div>

2). Attribute directives - 
These modify the appearance or behavior of an existing element (but don’t add/remove it).

eg :- ngClass → adds/removes CSS classes
      ngStyle → applies inline styles

code eg :- <p [ngStyle]="{color: 'red'}">Hello</p>      

3). Component directives - 
These are actually the most common kind of directive.
A component is a directive with a template.
It controls a section of the UI.
eg :- <app-header></app-header>

4). Custom Directives
You can also create your own directive.

-- Bash command
ng generate directive highlight

-- TypeScript
import { Directive, ElementRef } from '@angular/core';

@Directive({
  selector: '[appHighlight]'
})
export class HighlightDirective {
  constructor(el: ElementRef) {
    el.nativeElement.style.backgroundColor = 'yellow';
  }
}

-- Usage
<p appHighlight>Highlighted text</p>
